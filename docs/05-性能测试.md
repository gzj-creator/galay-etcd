# 05-性能测试

本页只保留真实 benchmark 入口、真实命令和当前结果状态。

## 1. Benchmark 真相来源

| 项目 | 当前值 |
| --- | --- |
| 源文件 | `benchmark/B1-etcd_kv_benchmark.cc` |
| target | `B1-EtcdKvBenchmark` |
| 客户端类型 | 同步 `EtcdClient` |
| 工作模型 | 每线程一个客户端实例 |
| 模式 | `put` 或 `mixed(put+get)` |

## 2. 命令行参数

源码中的参数顺序是：

1. `endpoint`
2. `threads`
3. `ops_per_thread`
4. `value_size`
5. `mode`：`put` 或 `mixed`

推荐命令：

```bash
./build/benchmark/B1-EtcdKvBenchmark http://127.0.0.1:2379 8 500 64 put
./build/benchmark/B1-EtcdKvBenchmark http://127.0.0.1:2379 8 500 64 mixed
```

环境变量：无。

## 3. Benchmark 实际统计项

`B1-EtcdKvBenchmark` 当前会输出：

- `Endpoint`
- `Mode`
- `Workers`
- `Ops/worker`
- `Value size`
- `Total ops`
- `Success`
- `Failure`
- `Duration`
- `Throughput`
- `Latency p50`
- `Latency p95`
- `Latency p99`
- `Latency max`

## 4. 运行前提与注意事项

- 需要一个可写的 etcd v3 HTTP endpoint
- benchmark 会写入 `/galay-etcd/bench/<timestamp>/...`
- benchmark 目标当前不负责清理所有写入数据
- `mixed` 模式是 `put` 后立即 `get`，不是只读压测

## 5. 当前结果状态

截至 `2026-03-10`：

- 本次文档修复只重新验证了 benchmark target 的存在与可构建性
- 没有在本次修复中重新执行 benchmark 并发布新数字

因此：

- 旧文档中的吞吐或延迟数字一律视为历史样本
- 它们不是当前版本的“已复算验收结果”

## 6. 重新发布性能数字的最低要求

如果你要把新的 benchmark 结果写回文档，至少同时记录：

- 执行日期
- 完整命令
- etcd 版本
- 机器规格（CPU / 内存 / OS）
- 编译器与构建类型
- 原始输出全文

没有这些元数据时，请不要把数字写成“当前性能”。
